Frameworks & Libraries: Much Faster Development

How we stopped reinventing the wheel.

David Heinemeier Hansson - Creator of Ruby on Rails [5]

John Resig - Creator of jQuery [6]

What Is A Framework

At the start, every application would be built from the ground up by the developers. This meant they would have to implement things like form validation, database connections, routes, a templating engine, etc. In those days there was no application layout or structure that everyone was using so every website looked very different to each other. The web was growing at a very rapid pace but it was happening at a time when the tools required to help support this growth had not yet evolved.

So frameworks and libraries came to be. A framework is a set of code that gives a standard way of structuring an application and providing common functionality. Instead of writing the basic application structure from scratch, a developer can use a framework and it will take care of less interesting tasks like routing, templating, database connections and security. A library on the other hand focuses on a particular aspect of programming. Libraries like jQuery make programming within a framework more simple and uniform.

Backend Frameworks

Structure For Servers

One of the most significant changes to the back end of a web application was brought about by the Ruby on Rails framework. Released in 2004 by David Heinemeier Hansson (DHH), the design principles of the Ruby on Rails framework have had lasting effects on the design of modern frameworks. The two principles that underpin the design of the Ruby on Rails framework are convention over configuration and don’t repeat yourself. Convention over configuration is a principle that eliminates the need to configure individual components of an application with knowledge of the application as a whole.

Django, a Python web application framework, was first released in 2005, while Laravel, a PHP web application framework, was first released in 2011. Despite being written in a different language than Rails, Django and Laravel were built with the same underlying principles in mind, including things like a clean application structure, tools to help with common tasks and an ecosystem of developers that build, release and maintain plugins and extensions to build web applications quickly. The application structure of web applications was defined and introduced with Rails, and it’s known as MVC (Model View Controller).

MVC defines three major parts in a web application: Model, View and Controller. The Model is responsible for data. The View is responsible for presentation. The Controller acts as an intermediary and receives input from the View and sends data to the Model.

Frontend Frameworks

From Basic Scripts To Full Applications

John Resig would release jQuery in 2006. Without jQuery or any other library writing JavaScript code was a chore. A chore in the most miserable sense of the word. To write code that worked across Internet Explorer, Firefox and Safari would take on average about 20 conditionals, work arounds and patches. jQuery would get rid of all that and at one point in time jQuery would be used on over 70% of all websites on the planet.

The complexity of modern web applications meant that even with the power of jQuery, other libraries were still required to maintain a robust system. In 2010, Google decided to make AngularJS, the first full frontend framework, responsible for managing application state, handling routing and keeping everything synchronized. In 2013, Facebook decided to release React, which is a component library. Rather than directly manipulating the DOM, like with jQuery, React uses a “virtual DOM” which is just a memory copy of the DOM.

Vue.js was started by Evan You who is a former member of the AngularJS team at Google and was released in 2014. Vue.js brings together the best qualities of AngularJS and React with a more approachable system for new developers.

Timeline of Events

2004

  • David Heinemeier Hansson releases Ruby on Rails, introducing convention over configuration and the MVC pattern to web development
  • Rails ships with a full set of tools out of the box, making it possible to build a working web application in minutes

2005

  • Django is released for Python, bringing Rails-style structure and rapid development to the Python community

2006

  • John Resig releases jQuery, solving the browser compatibility nightmare that had plagued JavaScript developers for years

2009

  • Node.js enables JavaScript on the server, opening the door for JavaScript-based backend frameworks
  • Express.js is released shortly after, becoming the most popular Node.js web framework

2010

  • Google releases AngularJS, the first major frontend framework for building full single-page applications in the browser

2011

  • Bootstrap is released by Twitter, giving developers a ready-made CSS framework for building responsive layouts

2013

  • Facebook releases React, introducing component-based UI development and the virtual DOM

2014

  • Evan You releases Vue.js after working on AngularJS at Google, offering a progressive framework that is easier to adopt
  • Vue gains rapid adoption due to its gentle learning curve and flexible architecture

2016

  • Angular 2 is released as a complete rewrite of AngularJS, switching to TypeScript and a component-based architecture
  • Next.js is released, bringing server-side rendering to React applications and blurring the line between frontend and backend
